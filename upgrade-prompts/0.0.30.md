# Upgrade to devrouter 0.0.30

## What changed

Devrouter now ships `devrouter-process ensure`, a Linux devcontainer helper for locked, fingerprinted, process-group-owned application startup. Generated devcontainers install the pinned Devrouter package and use this helper instead of carrying their own PID, locking, and restart implementation. HTTP readiness and the single container-recreate budget remain in `devrouter workspace ensure`.

## Steps

1. Install `@devrouter/cli@0.0.30` on the host and bump `.devrouter.yml` to `devrouter.version: 0.0.30`.
2. If the repository uses the managed scaffold, run `devrouter repo devcontainer write --dry-run --json`, review the plan, then run `devrouter repo devcontainer write --yes`.
3. For a custom devcontainer, install the pinned package in its image and replace repository-specific background-process lifecycle code with one `devrouter-process ensure` call. Keep only the application's command and environment setup in the repository.
4. Rebuild or recreate the devcontainer so the helper is installed in the image.
5. From a linked worktree, run `devrouter workspace ensure .` twice and confirm the second run reuses both the container and managed process.
